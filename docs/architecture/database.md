# Database

## Current Models
- User. Account identity and role.
- ApiKey. Encrypted exchange credentials.
- Strategy. Strategy configuration and risk parameters.

## Relationships
- User 1 -> N ApiKey
- User 1 -> N Strategy

## Planned Models (MVP)
- MarketUniverse. Base currency, filter rules, whitelist, blacklist, and auto-exclude rules.
- SymbolGroup. A named list of symbols derived from the universe.
- Bot. Running instance of a strategy and its execution mode.
- BotStrategy. Links bots to strategies and symbol groups.
- Position. One open position per symbol at a time.
- Order. Orders placed or simulated.
- Trade. Executed fills and PnL snapshots.
- Signal. Generated signal history.
- BacktestRun. Metadata for each backtest.
- BacktestTrade. Trades generated by backtesting.
- BacktestReport. Aggregated metrics for a run.
- Log. Audit trail for system and trading actions.

## Data Rules
- User IDs use cuid.
- Other entities use uuid.
- createdAt and updatedAt exist on core tables.
- Strategy configuration is stored as JSON for fast iteration.
- Strategy-to-symbol mapping is explicit through bot-to-group relations.

## Runtime Cardinality Contract (V1)
- User `1 -> N` Bot.
- Bot `N <-> N` SymbolGroup via BotStrategy rows.
- SymbolGroup `1 -> N` BotStrategy rows.
- BotStrategy is the runtime binding unit: one row binds exactly one bot, one symbol group, and one strategy.
- AI assistant entities are user-scoped and support User `1 -> N` Assistant (planned extension).

## Planned Extensions (After MVP)
- Strategy format versioning for import/export.
- Soft deletes for user-owned entities.
- Per-entity ownership checks enforced in API.
- Encryption at rest for API credentials.
